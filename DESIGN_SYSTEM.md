# D&D Toolbox — Дизайн-система

Единый визуальный язык для хаба и всех его модулей (текущих и будущих).
Источник правды. Любой новый модуль обязан использовать токены и компоненты отсюда, а не вводить свои цвета/шрифты.

Фиделити: **high-fidelity** — значения финальные, переносить точно.

---

## 1. Принципы

1. **Тёмное фэнтези по умолчанию, светлый пергамент как альтернатива.** Две темы: `dark` (Obsidian) и `light` (Vellum).
2. **Золото — общий мост.** Акцент по умолчанию золотой; он перекрашивает «хром» (кнопки, активные пункты, рамки готового), но не контент.
3. **У каждого модуля свой цветовой тег** (иконка/чип). Тег постоянен и не зависит от темы — так модули сохраняют узнаваемость.
4. **Линейные иконки, без эмодзи.** Контур 1.6, `currentColor`.
5. **Серьёзная типографика:** гравированный антиква Cinzel для бренда/заголовков + чистый гротеск Public Sans для UI.
6. **Минимализм и ритм.** Никакого «дата-слопа», градиентного шума, лишних рамок с цветным бордером слева.

## 3. Акценты (настраиваемый цвет)

Пользователь выбирает акцент. У каждого — пара значений (для тёмной/светлой темы, чтобы держать контраст).

| id | Имя | dark | light |
|---|---|---|---|
| `gold` (по умолч.) | Золото | `#c9a84c` | `#9a7421` |
| `crimson` | Багрянец | `#d06a52` | `#b3402a` |
| `teal` | Бирюза | `#45a3b3` | `#1f7d8c` |
| `violet` | Аметист | `#9b80d8` | `#6b4fb0` |
| `emerald` | Изумруд | `#4fb487` | `#2a7d56` |
| `copper` | Медь | `#c98a4f` | `#9c5f24` |

**Производные от акцента (`acc` = выбранное значение для текущей темы):**
```
accent         = acc
accentBright   = lighten(acc, light ? 0.14 : 0.28)   // ховеры, яркие штрихи
glow           = rgba(acc, light ? 0.13 : 0.16)      // подложка активного/«готов»
line           = rgba(acc, light ? 0.34 : 0.26)      // акцентная hairline (активные/готовые)
```
Хелперы `lighten`/`rgba` — см. `design-reference/parts.jsx`.

> Правило: нейтральная линия = `line2`, акцентная (активный пункт, рамка «готового» инструмента) = `line`.

## 6. Отступы, радиусы, тени

**Радиусы:** карточки `14–16`, плитки/строки `12`, поля и кнопки `8–9`, чипы/иконки `8–11`, пилюли/теги `100`.

**Spacing (px):** шаг сетки 4. Частые gap: `6, 8, 9, 14, 16, 18, 22`. Паддинг страницы: `34 48`. Паддинг панели/сайдбара: `18–22`.

**Тени:**
- ховер карточки: `0 14px 34px rgba(0,0,0,.34)` + `0 0 0 1px var(--line)`
- featured-карточка: `0 10px 30px rgba(0,0,0,.3)`
- крупное превью: `0 18–20px 50–60px rgba(0,0,0,.4–.45)`

**Переходы:** `120–160ms ease`. ⚠️ **Не анимируй `border-color` к ключевому слову `transparent`** — в некоторых браузерах переход «застревает» на исходном цвете. Анимируй `transform`/`box-shadow`/`background`, а смену цвета бордера делай мгновенной (выявлено и исправлено в прототипе).

## 7. Иконки

Линейные, viewBox `0 0 24 24`, `fill:none`, `stroke:currentColor`, `stroke-width:1.6`, скруглённые концы/стыки. Набор и пути — `design-reference/icons.jsx`.

Доступны: `character, token, npc, loot, tavern, city, dice` (инструменты); `gear, search, plus, grid, chevron, wand, book, pin, bolt, sun, moon, check, back, droplet, lock, lockOpen, reroll` (UI). Новые модули добавляют свою иконку в общий набор.

## 8. Компоненты (контракт)

Все берутся из дизайн-системы, не дублируются в модулях.

- **`Icon({name,size,stroke})`** — единый рендер иконок.
- **`BrandMark`** — гексо-логотип хаба.
- **`Button`** — `primary` (заливка `accent`, текст `onAccent`) и `secondary` (прозрачный, бордер `line2`, текст `text`). Радиус 9–10, паддинг `8–12 × 15–22`.
- **`ToolCard`** — три варианта: `feature` (готовые инструменты, крупная), `tile` (сетка), `row` (список/рейл). Содержит чип-иконку, имя, описание, статус-тег.
- **`StatusTag`** — «Готов» (акцентный) / «Скоро» (приглушённый).
- **`AppShell`** — каркас модуля: сайдбар (сгруппированная навигация) + верхняя строка (хлебные крошки + действия) + рабочая область.
- **`AppearanceControls`** — переключатель темы (☾/☀) + поповер выбора акцента.
- **`ToggleGroup`** — одиночный выбор, два варианта: `buttons` (по умолчанию —
  отдельные кнопки с бордером, как пол/длина в Character Forge) и `segment`
  (компактный сегмент-контрол в общем боксе: контейнер `raised` + `line2`,
  радиус 9, паддинг 3; активная кнопка — `glow` + `accent` + inset-кольцо
  `line`; как достаток/поселение/язык в Tavern Builder).
- **Формы:** `Field`, `Select`, `Slider` — единый вид полей (лейбл 10.5px, поле на `raised`, бордер `line2`, радиус 8–9).

Эталонные реализации — `design-reference/parts.jsx` и `screens.jsx`.

## 9. Реализация токенов в продакшене

В прототипе палитра прокидывается пропом `p`. В реальном коде **перейди на CSS-переменные** — это чище и быстрее:

```css
:root[data-theme="dark"]  { --ds-bg:#0f0d0a; --ds-panel:#171310; /* … */ }
:root[data-theme="light"] { --ds-bg:#ece3cf; /* … */ }
:root { --ds-accent:#c9a84c; --ds-on-accent:#14100a; --ds-glow:rgba(201,168,76,.16); --ds-line:rgba(201,168,76,.26); }
```

`ThemeProvider` ставит `data-theme` на `<html>` и пишет `--ds-accent*` из выбранного акцента (через `makePalette`). Компоненты читают `var(--ds-*)`. Тема и акцент сохраняются в `localStorage` (`ddtb_theme`, `ddtb_accent`).

## 10. Чек-лист нового модуля

- [ ] Зарегистрирован в реестре (`manifest`: id, name, description, group, icon, accent-тег, status).
- [ ] Иконка добавлена в общий набор `icons`.
- [ ] Использует только токены/компоненты дизайн-системы (никаких локальных хексов, кроме функциональной графики вроде рамок токенов).
- [ ] Работает в обеих темах и со всеми акцентами.
- [ ] Поля/кнопки/карточки — из общих компонентов.
- [ ] Состояние модуля переживает перезагрузку (`localStorage`), если это уместно.
